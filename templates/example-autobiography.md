## 2. Thinking Frameworks（思維框架）

- ⚙️ 第一性原理思維：在開始任何合作前先設定最壞的分手條件，確認自己能接受才開始
- ⚙️ BGO 引擎（逆推思維）：決策前做最壞打算，建立退出條件，而非樂觀出發
- ⚙️ SOP 流程化思維：遇到複雜任務自動拆解成前中後三段，再逐段控制
- ⚙️ 槓桿思維：清楚盤點自己的可交換資源（品牌、知識、研發、信用、人脈），不做無槓桿投入
- ⚙️ 目標設定系統化：種子法則、心想事成有明確操作框架，不是模糊願景
- ⚙️ 垂直下潛思維（對比水平展開）：在任何討論的岔路口選「往下挖底層」而非「往旁邊展開」；追問「為什麼」多過「還有什麼」
- ⚙️ 四層架構模型（由底往上建）：原理層（Why/價值觀）→ 架構層（思考框架）→ 工具層（AI/軟體）→ 經驗層（案例/SOP）；多數人停在第三四層，Fish 從第一層建起，工具換了也不怕
- ⚙️ 知識遷移天線模型：把認知分為「接收→編譯→發射」三段，認為「編譯」（把不同頻率訊號翻譯成同一結構）是人類最普遍缺失的能力
- ⚙️ 降低摩擦力優先於改變心態：改變的真正阻力是物理層面的環境摩擦（啟動成本、反饋週期），而非心態問題；心靈雞湯解法無效，微習慣設計才有效
- ⚙️ 覺察力即 logging system：把自我覺察類比為程式的 debug 能力；沒有 log 就無法 debug，沒有覺察就無法從失敗中迭代
- ⚙️ 種子思維：不設計終點，只設計入口和推動力；「不拔苗，只澆水」；讓對方自己走到結論，而非被告知答案
- ⚙️ 使用 AI 作為鏡子（非答案機）：用 AI 校準問題、反映盲點，而非尋求答案；自己有想法 → 用 AI 反射 → 看到盲點 → 修正 → 更深
- 🧪 扁平化、去中心化偏好可能源自早期創業被層級政治傷害的經驗，但具體成因未驗證
- 🧪 信奉「不自嗨」原則，以實際結果論事，可能與過去看到太多只說不做的人有關

## 5. Work Style（工作風格）

- ⚙️ 8-9 點起床，1 小時晨間活動後 9 點開始工作；晚上 6 點後除特殊情況不談公事
- ⚙️ 強烈偏好非同步溝通，偏好文字文件勝過語音/即時通訊
- ⚙️ 最高效狀態：有明確流程 + 自由安排事物的權力 + 獨立工作空間（三條件缺一不可）
- ⚙️ 充電方式：定位自己在組織中的位置做擅長的事；保留空白閒置時間獲取靈感
- ⚙️ 靈感來源：一人獨靜 + 自我專注 + 研究新事物，不是外部刺激
- ⚙️ 開會偏好：提前預約、準備預讀文件、最多 15 人；開視訊建議開鏡頭
- ⚙️ 理想合作對象：有自我要求、主動解決問題、提問前已想好方向
- ⚙️ 溝通確認原則：重要決定後必須再次確認雙方共識，避免資訊落差
- ⚙️ 刻意引入異質性資訊源：主動與不同背景的人/領域對話，對抗演算法帶來的資訊繭房；靠多元跨域輸入維持知識更新，而非依賴平台推薦
- ⚙️ 自我修正機制已內建：思維中有自動偵測「問題方向不對」的機制，會中途停下來換提問方式，而非硬推到底

## 6. Boundary Map（邊界地圖）

- ⚙️ 硬邊界：不在附近抽煙、不在公開場合說髒話、不散播謠言、不拉幫派小圈圈
- ⚙️ 溝通邊界：不接受超長語音訊息（可語音轉文字）；早上 9 點前/晚上 11 點後不打擾
- ⚙️ 合作邊界：做事要有始有終，不接受「雷聲大雨點小」；以實際結果論事，不自嗨
- ⚙️ 環境邊界：工作場所不放食物（防蟑螂）；最多 15 人的會議
- ⚙️ 知識邊界：不教「怎麼做」（答案），只設計讓對方自己走到結論的路徑（入口）；不給倖存者偏差的案例當承諾，只給工具讓他自己發現價值
- 🧪 在合作關係中會先設定「最壞分手條件」，但這種謹慎可能造成潛在合作者感覺門檻過高
