# PLM 採訪問題集
> 設計原則：每個問題要能問出具體故事或決策，而非主觀感受陳述。
> 答案足夠後，PLM 依 extraction-framework.md 自動評分。
> 版本：v1.0 | 建立：2026-05-31 | 來源：從自傳特質反推

---

## 主題一：人格底層

1. 你有沒有在一段對話或讀一本書的過程中，突然「停不下來」，一定要追到底？當時發生了什麼、是什麼讓你停不下來？
→ [結構直覺 / 垂直下潛]

2. 最近一次你做了一件事之後，別人說「你怎麼想得到？」——你自己覺得那個想法是怎麼冒出來的？
→ [跨域遷移能力 / 結構直覺]

3. 你曾經在某件事失敗之後，反而繼續做得更認真、更深入，而不是放棄？那次是什麼情況？
→ [元認知緩衝層 / 失敗=Log]

4. 有沒有某個你一直覺得「很正常」的習慣或思維方式，後來才發現很多人做不到？當時你怎麼看待這件事？
→ [獨特性認知盲點 / 排他性特質]

5. 如果你的朋友要幫你寫推薦信，他最可能說「這個人跟別人不一樣的地方是——」，你猜他會說什麼？
→ [人格核心 / 自我認知]

## 主題二：思維與決策框架

1. 說一個你做過最「反直覺」的決定——別人都說不對，但你還是做了，後來怎麼樣？你當時怎麼想的？
→ [第一性原理 / 灰度思維 / BGO 逆推]

2. 你怎麼決定一件事「夠好了可以停」還是「還需要繼續做」？給我一個具體的例子。
→ [三問檢驗 / 熵減 / 血清素設計]

3. 你曾經把某個領域學到的東西，直接搬到完全不相關的地方用？那是什麼？你怎麼發現可以這樣用？
→ [底層遷移 / 知識遷移天線模型]

4. 面對一個你完全不懂的新領域，你的第一步是什麼？給我一個你最近的例子。
→ [Inside-out / 四層架構模型]

5. 你最近一次改變對一件事的看法，是因為什麼？你是先有「感覺不對」還是先有「邏輯推導出不對」？
→ [元認知 / 訊號噪音辨別 / 自我修正機制]

## 主題三：人生軌跡與關鍵轉折

1. 你的人生有沒有一個時刻，你覺得之前的「規則」全部失效了、必須重新想過自己要怎麼活？那是什麼事情？
→ [三觀崩塌後的重建 / 元認知緩衝層起源]

2. 你到目前為止做過最難的一個決定是什麼？難在哪裡？你怎麼下決定的？
→ [BGO 逆推 / 退出條件設計 / 人生時間線]

3. 有沒有一次「失敗」，你現在回頭看反而覺得是轉折點？當時你的心理狀態是什麼？
→ [失敗=Log / 元認知緩衝層 / 人生時間線]

4. 如果把你的職業生涯分成「之前」和「之後」兩段，那個分水嶺是什麼事？
→ [關鍵決策 / 人生時間線]

5. 現在做的事情和十年前想要做的事情，有多大的差距？那個落差是怎麼發生的？
→ [人生軌跡 / 使命演化]

## 主題四：專業能力地圖

1. 你做哪一件事的時候，時間特別容易「不見」——不是因為很痛苦，而是進入了狀態？那件事是什麼？
→ [天賦領域 / 技師原動力]

2. 你自己學過最快的一件事是什麼？你覺得你學那麼快的原因是什麼？
→ [個人學習技術 / 底層遷移 / 方法論]

3. 你現在做的幾件事裡，哪一件最難被別人複製？你覺得為什麼？
→ [核心能力 / 排他性專業]

4. 你有沒有在不需要正式學習的情況下，就做出了一個讓你自己也驚訝的東西？那個過程是怎麼發生的？
→ [自學能力 / 跨域整合 / AI 協作能力]

5. 你覺得你的哪個能力，現在還沒有被完整地用到？為什麼沒用到？
→ [潛力領域 / 槓桿缺口]

---

## 主題五：工作邊界與合作模式

1. 你和合作夥伴相處過最不舒服的一次是什麼情況？那次你做了什麼？你覺得對方哪裡讓你不舒服？
→ [邊界地圖 / 合作邊界 / 理想合作對象]

2. 你最理想的一天工作狀態是什麼樣子？從早到晚——誰都不來打擾你那天，你會怎麼分配時間？
→ [工作風格 / 充電方式 / 最高效狀態]

3. 你接過最「不適合自己」的工作或合作是什麼？你怎麼判斷它不適合？後來怎麼處理？
→ [邊界地圖 / 課題分離 / 合作退出條件]

4. 你怎麼判斷一個人「值得合作」？你有沒有一個標準或測試？
→ [合作邊界 / 最壞分手條件設計]

5. 有沒有哪種工作方式或合作模式，你試過之後決定「這輩子不再用」？那次發生了什麼？
→ [硬邊界 / 合作邊界 / 人格底層]

## 使用說明

### 什麼情況問哪組

| 情況 | 優先問的主題 |
|------|-----------|
| 核心區塊資料完全空白 | 主題一（人格底層）→ 主題五（工作邊界） |
| 思維框架資料不足（< 2 個來源） | 主題二（思維與決策框架） |
| 人生時間線只有表層職業記錄 | 主題三（人生軌跡與關鍵轉折） |
| 專業地圖缺乏深度描述 | 主題四（專業能力地圖） |
| 工作風格/邊界資料不足 | 主題五（工作邊界與合作模式） |

### 怎麼判斷問夠了

對每個主題，每一題的回答需同時滿足：
1. **有具體事件**：不是「我覺得我是...」，是「有一次...」
2. **有決策過程**：說明「當時為什麼這樣做」
3. **有可萃取特質**：題目後面標注的 `→ [類型]` 都能從回答中找到對應的行為模式

每個主題至少 2 題有完整故事 → 可繼續萃取，不需要把同主題的 5 題全問完。

### 追問原則

若受訪者的回答太短或過於抽象，優先追問：
- 「那時候你的第一個念頭是什麼？」
- 「你最後怎麼決定的？」
- 「如果重來一次你還會這樣做嗎？」

不要追問「你覺得自己是什麼樣的人？」——這種問題會得到主觀感受，不是可萃取的事實。
